Js隐式类型转换需要注意的地方

发表于 2017-07-30 | 更新于 2017-07-30 | 分类于 JavaScript| 阅读量

加号

  • 等式中如果有字符串则在字符串之后的类型调用String()方法转化成字符串(null与undefined没有toString()方法），最后结果一定是字符串，运算方向是从左到右的,注意对象与数组的处理方式
  • 等式中如果没有字符串，如果有复杂类型，先将复杂类型调用valueOf()取得返回值,如果返回值不能转化成数值（即返回NaN)，调用toString()将返回值转化成字符串，再按照1的运算方式。没有有复杂类型则将简单类型隐式转换为数字即可。
  • 注意undefined 转化成数字是NaN
  • 特殊注意

乘，除，取余，减

  • 简单类型则会隐式转换为数字，对于字符串转成数字，如果字符串不是纯数字就会转成NaN。

  • 有复杂类型，先将复杂类型调用valueOf()取得返回值,如果返回值不能转化成数值，调用toString()将返回值转化成字符串，再由字符转化成数字，最后从左到右计算

比较运算符部分 > < >= <= ==

  • 数字vs其他，其他转化为数字
  • 布尔值vs其他，布尔值转数字
  • 字符串vs字符串，按unicode依次比较(大写字母总是在小写字母之后)
  • 对象vs数字，对象vs字符串，对象首先尝试valueOf()，再尝试toString()【日期类只调用toString()】，目的都是将对象转化为原始值，再进行比较。
  • 特殊注意
    • 如果其中一个操作数是NaN，那么总是返回false(NaN和NaN是不相等的)、
    • null == undefined，为true
    • null、undefined既不与0相等，也不与false相等
