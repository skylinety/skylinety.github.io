chapter 2. let和const命令

1.let const

a. let
  • let 只在申明的代码块内部有效
  • 所在作用域形成暂时性死区，不存在变量提升，需要声明后使用，不可重复声明
b. const
  • const 用于声明常量，声明即须初始化
  • 所在作用域形成暂时性死区，不存在变量提升，需要声明后使用，不可重复声明
  • 将一个对象声明为常量时，是指常量名指向的栈中保存放的堆地址不变，而不是指针指向的堆内存中数据不变
c. 代码

2. 变量声明

a. 要点
  • es6 提供6总变量声明方式，包括let const var function class import
  • var function 在全局作用域声明仍属于全局，而let const class 不再属于全局。
b. 代码

chapter 3. 变量的解构赋值

1. 数组解构赋值

a. 要点
  • 按照位置对应关系对变量赋值，属于“模式匹配”
  • 等号右边需要是可遍历结构（具备Iterator接口，即需要具有Symbol.iterator属性），否则报错
  • 解构部分不成功时，不成功的变量值是undefined
  • generator对象具备Iterator接口
  • 允许指定默认值，默认值只有对应位置是undefined时才会生效；如果默认值是表达式，表达式是惰性求值的，即对应位置成功解构则不执行
b. 代码

2. 对象解构赋值

a. 要点
  • 对象是按照属性名对应关系来进行变量赋值
  • 对象解构赋值是先找到同名属性，然后赋值给对应变量。双方的键不会被赋值，真正赋值的是值
  • 也可以指定默认值，使用默认值的条件同样是赋予的值严格等于undefined
  • 对于已声明的变量解构，需要注意解析引擎对‘{’解析成代码块
b. 代码

3. 其他解构赋值

a. 要点
  • 解构赋值的基本规则是只要有变动值不是对象，就现将其转化为对象。（null和undefined无法进行转换，除外，Boolean、Number、String是基本包装类型）
  • 函数参数解构赋值
b. 代码

4. 圆括号问题

a. 要点
  • 变量声明时，模式匹配尽量不使用圆括号
  • 不要将整个模式放在一层圆括号中
